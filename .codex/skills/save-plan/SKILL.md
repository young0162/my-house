사용자가 이 스킬을 호출하면 대화에서 논의된 계획을 파일로 저장합니다.

## 처리 순서

1. **계획 내용 파악**: 대화 히스토리에서 가장 최근에 논의된 계획, 구현 전략, 기능 기획, 아키텍처 제안 등을 찾습니다.

2. **기획명 결정**: 계획의 주제에서 간결한 이름을 추출합니다.
   - 한국어 또는 영어, 단어 사이는 언더스코어(`_`) 사용
   - 예: `로그인_기능`, `결제_플로우`, `상품_상세페이지`, `DB_스키마_설계`
   - 이름이 너무 길면 핵심 키워드 2-3개로 압축

3. **파일명 생성**: 오늘 날짜 + 기획명 조합
   - 형식: `YYYYMMDD_기획명.md`
   - 예: `20260602_로그인_기능.md`
   - 날짜는 context의 `currentDate`를 사용하거나 시스템 날짜를 사용

4. **디렉토리 확인**: 프로젝트 루트의 `docs/plan/` 폴더가 없으면 생성

5. **파일 저장**: 계획 내용을 마크다운 문서로 저장

6. **저장 확인**: 저장된 파일 경로를 사용자에게 알려줌

## 저장 파일 형식

```markdown
# [기획명]

> 저장일: YYYY-MM-DD

## 개요
[이 계획의 간단한 요약]

[대화에서 논의된 계획 전체 내용 — 섹션 구조 유지]
```

논의된 내용에 충실하게 작성하고 임의로 범위를 추가하거나 삭제하지 않습니다. 계획에 이미 섹션 구조가 있다면 그대로 유지합니다.

## 예시

사용자가 로그인 기능 구현 계획을 논의한 후 "show plan"을 입력하면:

- 기획명: `로그인_기능`
- 저장 경로: `docs/plan/20260602_로그인_기능.md`
- 완료 후 알림: "계획이 `docs/plan/20260602_로그인_기능.md`에 저장되었습니다."
